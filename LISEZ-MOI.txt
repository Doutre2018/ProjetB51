1. Programme:
Le présent logiciel en est un de gestion de logiciels de type SAAS. 
Il s'inscrit dans le programme de cours B51 où les étudiants 
sont amenés à concevoir le gestionnaire de logiciels ainsi que 
les logiciels (modules) qui le supportent.

2. Outils:
- Python
- Eclipse
- SQLite

II. MODE D'EMPLOI
===============================
Pour accéder à la version client, ouvrir gestpro_serveur.py 
dans le répertoire "2018_GestPro_serveur". Ensuite, ouvrir 
gestpro_client_main.py dans le répertoire "2018_GestPro_client".

- À l'ouverture du SAAS, si vous n'avez pas d'utilisateur:
		-Cliquez sur "Nouveau Client"
		-Entrez votre nom d'usager
		-Entrez votre mot de passe
		-Confirmer le mot de passe (ne copier/coller pas le mot de passe d'au-dessus)
		-Choisissez la compagnie dont vous êtes le représentant.
- Pour vous connecter:
		-Entrez le nom d'utilisateur, votre mdp, votre compagnie ainsi que l'adresse IP du serveur.

DANS LE GESTIONNAIRE:
	POUR QUITTER LE PLEIN ÉCRAN: ACCÉDER AU MENU EN HAUT, OPTION «AFFICHAGE», «FULL SCREEN»

CHOISSISEZ UN MODULE:
- Mandat :
	Innaccessible pour le moment (l'interface utilisateur n'est pas fonctionnelle)
-SCRUM :
	(BD pas connectée)
	Cliquez sur le bouton ajouter en dessous des zones de texte pour ajouter le contenu.
	Il n'y a pas de persistance de données.
-Analyse textuelle:
	Pour ajouter des mots/verbes/adjectifs, cliquez sur les petits "+" et ajoutez le contenu dans
	la nouvelle fenêtre.
	Il y a persistance des données, mais pas du tout au point
-Cas d'usage:
	Cliquez sur les cas à gauche pour faire afficher leur contenu dans les catégories à droite.
	Cliquez sur les boutouns en dessous pour ajouter du contenu.
	Il n'y a pas de persistance de données.
- La Maquette :
	Pour ajouter des formes/textes, cliquez sur les objets dans le menu.
	Pour changer la couleur d'un objet, ouvrez le menu défilant en haut pour faire apparaitre la palette de couleur
	Pour ajouter du texte, cliquez sur le grand "T"; le nouveau texte apparaitra au milieu du canvas. 
		Ce texte est éditable en double cliquant dessus. Entrez le nouveau texte dans la nouvelle fenêtre.
	Il n'y a pas de persistance de données.
- CRC
	Le module s'ouvre avec le nom des cartes CRC
	En cliquant sur une carte, ça ouvre une fenêtre dans laquelle on peut entrer les informations pertinentes.
	On peut ajouter des cartes, ajouter du contenu.
	On peut supprimer des cartes (ce qui les supprime en BD aussi)
	On ne peut PAS modifier le contenu des cartes existantes (prochaine implantation)
	Il y a persistance des données (mais pas propre au "projet")
-Budget
	Pas implanté
-Tchat
	Interface est terminée, mais il n'y a pas de persistance de données et l'utilisateur n'est pas lié au module.
	Il n'y a pas de persistance de données.
-Modélisation de données
	Pour ajouter une nouvelle classe, cliquer sur «Créer table»
	Dans la nouvelle fenêtre, entrez le nouveau nom de la table.
	Cliquez sur «Ajouter champs» pour ajouter un nouveau type de champ (données)
	Pour ajouter cette table au projet, cliquez sur «Ajouter».
	Il n'y a pas de persistance de données.
-Terlow
	On peut ajouter des colonnes, dans lesquelles on ajoute des classes.
	Il n'y a pas de persistance de données
